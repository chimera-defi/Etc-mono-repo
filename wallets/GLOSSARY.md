# Glossary of Wallet Terms

This glossary defines core terms used across Wallet Radar comparisons.

## Terms

**EOA (Externally Owned Account)**  
Standard wallet address controlled by a private key. Most wallets are EOAs by default.

**Account Abstraction (EIP-4337)**  
Smart contract wallets that enable features like session keys, gas sponsorship, and batch transactions.

**EIP-7702**  
Allows EOAs to temporarily act like smart contract accounts for batched actions and sponsored gas.

**Multi-sig**  
Wallets that require multiple approvals to move funds (e.g., Safe).

**Transaction Simulation**  
Previews what a transaction will do before signing (token changes, approvals, risks).

**Scam/Phishing Detection**  
Heuristics or databases that warn on suspicious contracts or approvals.

**Hardware Wallet**  
Dedicated device that stores private keys offline (e.g., Trezor, ColdCard).

**Software Wallet**  
Browser or mobile wallet that stores keys in software (e.g., Rabby, Rainbow).

**On-ramp / Off-ramp**  
Services that convert fiat to crypto (on-ramp) or crypto to fiat (off-ramp).

**Canonical URL**  
The primary URL that search engines should index for a page.

**JSON-LD**  
Structured data format used for SEO schemas (e.g., FAQPage, Product).
