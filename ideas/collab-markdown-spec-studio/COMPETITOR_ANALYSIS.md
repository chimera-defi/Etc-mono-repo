## Competitor Analysis (Spec-to-Ship Workspace)

## Categories
1. Collaborative docs suites (Notion/Google Docs/Coda)
2. Collaborative markdown tools (HackMD/HedgeDoc)
3. AI writing/assistant tools
4. Repo scaffolding/project generators

## Competitive Snapshot

### Collaborative Docs Suites
- Strengths: distribution, polish, team habits.
- Weaknesses: weak spec-to-code execution flow, limited patch-governance semantics.
- Opportunity for us: workflow-native spec-to-ship pipeline.

### Collaborative Markdown Tools
- Strengths: markdown-native collaboration simplicity.
- Weaknesses: limited AI governance and execution handoff capabilities.
- Opportunity for us: structured agent patch review + build artifact exports.

### AI Writing Tools
- Strengths: strong generation speed.
- Weaknesses: weak multi-user merge/provenance controls.
- Opportunity for us: collaborative trust and auditability.

### Repo Scaffolding Tools
- Strengths: fast project bootstrapping.
- Weaknesses: weak upstream spec traceability.
- Opportunity for us: spec-linked generation and feedback loop.

## Positioning Statement
"Not a docs replacement; a spec-to-ship workflow system with human+agent governance."
