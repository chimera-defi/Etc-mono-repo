## Competitor Matrix (SpecForge / Spec-to-Ship Workspace)

Last updated: 2026-03-07

Scoring legend:
- `Strong`: first-class capability
- `Partial`: capability exists but not core or limited
- `Weak`: mostly absent

| Competitor | Category | Real-time Collaboration | AI Co-authoring | Patch Governance (accept/reject/cherry-pick) | Spec-to-Code Handoff | Notes |
|---|---|---|---|---|---|---|
| Notion (+ Notion AI) | Collaborative docs suite | Strong | Strong | Weak | Partial | Powerful docs + AI, but merge-safe agent patch governance is limited. |
| Google Docs (+ Gemini) | Collaborative docs suite | Strong | Partial | Weak | Weak | Ubiquitous collaboration; limited spec-to-build workflow primitives. |
| Confluence (+ Atlassian Intelligence) | Enterprise docs/wiki | Strong | Partial | Weak | Partial | Strong enterprise distribution; less focused on spec-to-ship agent flows. |
| Coda (+ Coda AI) | Collaborative docs + automation | Strong | Strong | Weak | Partial | Strong structured docs/automation; weak section-level AI patch workflow. |
| HackMD | Collaborative markdown editor | Strong | Weak | Weak | Weak | Markdown-native collaboration; minimal governance and build handoff. |
| HedgeDoc | Open-source collaborative markdown | Strong | Weak | Weak | Weak | Good collaborative markdown base; little AI governance layer out-of-box. |

## Positioning Takeaway
1. Most incumbents are strong at collaboration and/or AI drafting.
2. Main gap remains governance + traceable spec-to-build handoff.
3. SpecForge should compete on controlled human+agent merge flow and execution readiness.
