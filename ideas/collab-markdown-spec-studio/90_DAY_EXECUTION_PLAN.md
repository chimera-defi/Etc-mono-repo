## 90-Day Execution Plan (SpecForge)

## Days 0-30: Discovery + Workflow Baseline
1. Conduct 15 interviews across founder/product/engineering roles.
2. Map baseline spec-to-first-commit workflows.
3. Recruit 5 design-partner teams.
4. Freeze MVP scope around collaboration + patch governance + depth gates + export.
5. Lock implementation defaults:
   - single TypeScript app + collaboration service
   - canonical editor JSON + derived block index
   - `block_id` patch targeting
   - GitHub OAuth for pilot users
   - curated benchmark corpus

Deliverables:
- workflow baseline report
- design partner cohort
- MVP scope freeze doc
- implementation-defaults memo

## Days 31-60: Pilot MVP Workflow
1. Run collaborative spec sessions with patch review flows.
2. Measure patch acceptance, trust, retention, quality, and speed.
3. Iterate UX friction points weekly.
4. Validate export bundle usefulness in real projects and curated `ideas/` examples.
5. Build and maintain golden fixtures for the default benchmark corpus.

Deliverables:
- pilot telemetry report
- patch trust analysis
- export bundle quality report
- benchmark corpus results

## Days 61-90: Decision + Phase 2 Gate
1. Compare outcomes against go/no-go scorecard.
2. Decide if curated repo generation enters active build scope.
3. If continue: freeze phase-2 spec and roadmap.
4. If continue: confirm the single-template generation boundary before implementation expands.

Deliverables:
- go/no-go decision memo
- phase-2 scope document
- implementation handoff package

## Resource Assumptions
- 1 product lead
- 1 full-stack/realtime engineer
- 1 AI/workflow engineer
