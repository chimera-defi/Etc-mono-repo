## Collaborative Spec Product Validation Plan

## Goal
Validate whether teams adopt this as a repeat workflow (not one-time novelty) and whether "spec -> repo" materially improves execution speed.

### Phase 0: Broad Discovery (Week 1)
1. Interview 15 teams:
   - founders/product leads
   - eng leads/tech leads
   - AI-heavy dev teams
2. Map current workflow:
   - where they write specs
   - how AI is used today
   - where handoff breaks

### Phase 1: Workflow Fit Test (Week 2-3)
1. Prototype with multiplayer markdown + patch approval UI.
2. Run 5 design-partner sessions on real projects.
3. Measure:
   - time to implementation-ready spec
   - patch acceptance rates
   - user trust in agent edits

### Phase 2: Outcome Test (Week 3-5)
1. Add export bundle and manual repo scaffold handoff.
2. Compare against control workflow (Notion/Docs + chat copy-paste).
3. Evaluate speed delta and quality outcomes.

### Phase 3: Repo Generation Test (Week 5-7)
1. Pilot one-click starter repo on 3 teams.
2. Track first-commit time and rework rate.
3. Decide if repo generation is core or optional module.

## Success Thresholds
1. >= 40% faster spec-to-first-commit cycle in pilot teams.
2. >= 35% agent patch acceptance in mature sessions.
3. >= 60% weekly return rate during active project windows.

## Kill Thresholds
1. Teams still default to incumbent docs for final specs.
2. Agent patch acceptance stays too low to create leverage.
3. Reliability/merge complexity undermines trust and retention.
