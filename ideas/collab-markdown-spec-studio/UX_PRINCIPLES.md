## UX Principles (SpecForge)

1. Make collaboration state obvious.
2. Make agent actions reviewable before apply by default.
3. Keep human intent primary; agent work is assistive, not authoritative.
4. Minimize context switching between chat, doc, and decisions.
5. Preserve traceability from requirement -> patch -> decision -> output artifact.
6. Show confidence and risk signals on generated edits.
7. Optimize for fast convergence, not maximal feature density.
8. Keep failure states explicit, recoverable, and low-drama.

## UX Constraints
1. Multi-user edits must feel stable under concurrency.
2. Diff/merge decisions should stay section-scoped, not whole-doc by default.
3. Recap and open-questions panel is mandatory at milestone boundaries.
4. Mobile is review-first in MVP; authoring-first on desktop.
