## Meta Learnings (Spec-to-Ship Workspace)

1. Strong naming and positioning matter early in crowded categories.
2. Separate collaboration value from execution value; connect them with explicit flow.
3. Add phase gates so complexity (repo generation) does not bloat MVP.
4. Keep concise executive docs for humans and deep docs for builders.
5. Multi-pass review logs improve clarity and iteration quality.
6. Use scorecards and financial/risk templates to convert idea docs into decision-ready assets.
7. Make "depth completion" explicit: do not stop until PRD/SPEC/risk/validation/economics are all present.
8. If detail is missing, the agent should ask targeted continuation questions instead of ending the run.
9. Always end major iterations with a concise recap to prevent idea drift.
10. Treat idea-development workflow as product surface area; this can be a core SpecForge feature.
